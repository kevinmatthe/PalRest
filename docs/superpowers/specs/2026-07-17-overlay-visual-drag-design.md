# 悬浮条视觉还原与 macOS 拖动修复设计

## 目标

将游戏内悬浮条还原为用户确认的 `480 × 76 px` 双层紧凑结构和“轻透冷青”视觉语言，同时修复 macOS 调整模式中无法拖动窗口的问题。

本次只修改桌面悬浮条及其拖动能力，不重做设置窗口，不改变 Go API、快照契约、轮询、频控计算或桌面生命周期语义。

## 已确认决策

- 视觉范围只包括游戏中的悬浮条，设置窗口保持当前结构。
- 固定结构为：左侧 `62 × 62 px` 小地图；右侧上层身份、状态、延迟和更新时间；右侧下层四项等宽计时；底部 `2 px` 进度线。
- 视觉采用“轻透冷青”：弱边界、柔和玻璃层次、克制阴影与较大圆角，数据优先，不使用管理后台式分组卡片。
- 正常状态使用冷青；临近限额使用琥珀；已达限制或休息中使用红色；离线或断连降低饱和度并保留最后有效数据。
- 不闪烁、不弹窗，不为状态变化改变整体布局。
- 调整模式不增加常驻拖动把手；进入后整条悬浮窗都可拖动，锁定后恢复鼠标穿透。

## 视觉结构

悬浮窗口继续使用 Tauri 配置中的 `480 × 76 px` 无边框透明窗口。React 根组件在该客户区内占满宽高，不通过 CSS transform 或内容尺寸扩大原生窗口。

```text
┌──────────────┬──────────────────────────────────────────────────┐
│              │ 玩家名称 · Lv.    在线       42 ms · 2s ago    │
│  62 × 62     ├──────────┬──────────┬──────────┬───────────────┤
│  局部小地图   │ 今日      │ 本周      │ 本周期    │ 频控剩余       │
│              │ 1h 24m   │ 7h 06m   │ 42m      │ 18m           │
└──────────────┴──────────┴──────────┴──────────┴───────────────┘
                  底部 2 px 策略进度线
```

布局要求：

- 根容器使用明确的两列网格，左列固定为地图尺寸，右列为 `minmax(0, 1fr)`。
- 右侧使用两行网格；身份行不因长名称或更新时间撑宽窗口。
- 四项计时保持等宽，标签和值均允许省略显示，但不能挤压相邻项。
- 缺少地图能力时右侧内容占满宽度；缺少计时能力时保留安静的底部留白或细分隔，不伪造数据。
- 小地图继续使用真实私网瓦片；加载失败只降级地图区域，不影响其他指标。

## 视觉语言

默认容器为深色半透明玻璃材质：

- 以深青黑为底，冷青只用于可交互状态、在线状态、延迟和轻微边缘光。
- 使用低对比细边框、单层内高光、柔和背景模糊和克制外阴影。
- 圆角比当前工业终端风格更柔和，但不做胶囊形，也不引入帕鲁专属装饰，以便未来其他游戏复用。
- 字体继续优先使用系统中文字体，数字采用等宽数字特性，避免引入网络字体。
- 纹理和网格仅限小地图内部；主信息区不使用重复斜纹或强烈战术刻度。

状态只替换语义色变量，不复制布局规则：

| 状态 | 表达 |
| --- | --- |
| 正常 | 冷青边缘、在线状态、延迟与进度 |
| 临近限额 | 琥珀边缘、状态、剩余值与进度 |
| 已达限制 / 休息中 | 红色边缘、状态、关键剩余值与进度 |
| 离线 / 断连 / 过期 | 整体降饱和，保留累计值和最后更新时间 |
| 调整位置 | 冷青虚线外框与居中拖动提示，不改变内容位置 |

遵循 `prefers-reduced-motion` 与 `prefers-contrast`；状态切换只允许短时颜色、边框和阴影过渡。

## macOS 拖动修复

当前拖动失败有两个已验证的原因：

1. Tauri capability 只允许事件订阅，没有允许 `start_dragging` 命令。`data-tauri-drag-region` 最终会调用该命令，因此请求被权限层拒绝。
2. 调整模式给根节点传入了布尔值 `true`。在当前 Tauri 版本中，这只允许直接点击根节点本身触发拖动；点击地图、文字或计时子节点不会继承拖动能力。

修复设计：

- 在桌面 capability 中增加 `core:window:allow-start-dragging`，不增加其他窗口权限。
- 调整模式将悬浮条根节点标记为 `data-tauri-drag-region="deep"`，使非交互子树中的任意点击都能触发原生拖动。
- 正常模式完全移除拖动属性，并继续由 Rust 设置鼠标穿透和不可聚焦。
- 调整模式继续由现有生命周期依次执行：显示窗口、关闭穿透、允许聚焦、聚焦、发布调整事件。
- 锁定继续先保存窗口坐标，再恢复穿透、可靠失焦并移除调整状态。
- 小地图在调整模式下不提供缩放或平移交互；整块区域优先用于拖动窗口。

## 组件与数据边界

- `OverlayBar` 只负责能力式结构、状态类名和拖动区域属性。
- `PalworldMiniMap` 继续负责真实瓦片、坐标投影和地图降级，不接管窗口拖动状态。
- 游戏 Adapter 继续提供持续时间格式化和总体 tone，不新增视觉布局职责。
- Rust 生命周期继续负责穿透、聚焦、显示、隐藏和位置持久化。
- Tauri capability 只开放深层拖动所需的单项窗口命令。

快照数据流保持不变：Go Provider → `overlay.snapshot/v1` → Tauri HTTP bridge → React poller → `OverlayBar`。本次不新增字段，也不在客户端重新推导频控状态。

## 错误与降级

- 地图瓦片错误不改变悬浮条尺寸或计时布局。
- 长玩家名、长状态文本和本地化时间标签必须在自身网格单元内截断。
- `stale`、`disconnected` 状态继续显示最后有效快照；没有快照时继续显示紧凑状态文案。
- 拖动命令失败不得改变已保存位置；用户仍可从托盘重新进入调整模式。
- 锁定时位置持久化失败继续按现有生命周期回滚，不伪装为成功。

## 测试与验收

### React / CSS

- 根容器严格适配 `480 × 76 px` 客户区，地图为 `62 × 62 px`。
- 右侧保持身份行和四项等宽计时行；长文本和能力缺失不破坏网格。
- 正常、警告、危险、离线、断连和调整状态使用预期变量与类名。
- 正常模式不存在拖动属性；调整模式根节点为 `data-tauri-drag-region="deep"`。
- 地图、文字和计时子节点均位于深层拖动区域内。
- `prefers-reduced-motion` 与高对比规则继续生效。

### Rust / Tauri

- capability 精确包含事件监听、取消监听和 `core:window:allow-start-dragging`，不开放额外窗口控制权限。
- 现有进入调整、锁定、位置持久化、穿透和可靠失焦测试继续通过。
- Rust 格式、Clippy 和无原生 Linux 系统依赖的单元测试全部通过。

### 构建与实机

- Overlay 前端完整测试和生产构建通过。
- GitHub Actions 的 Windows 与 macOS Tauri 构建通过。
- macOS 实机从菜单栏进入调整模式后，分别从地图、玩家名、计时格和空白区域拖动，窗口均跟随；锁定后所有区域恢复鼠标穿透。
- Windows 执行同一拖动与锁定烟测，防止共享属性修复造成平台回归。

## 不在本次范围

- 设置窗口视觉重做。
- 窗口尺寸、计时项目或小地图能力变更。
- 全局快捷键、独占全屏覆盖、游戏注入或内存读取。
- 新的后端字段、认证、插件系统或多游戏运行时加载。
